## 一、项目概述

Reverie 是一个极简风格的 AI 聊天 Web 应用，部署路径为 `/chat/`，后端服务运行在 `https://kdreamling.work`。前端使用 React + TypeScript + Vite 构建，设计语言以克莱因蓝（`#002FA7`）为核心色，追求深邃夜空 + 极简排版的视觉风格。

### 3. 路由配置（`src/App.tsx`）

```
BrowserRouter basename="/chat"
├── /login          → LoginPage
├── /               → ChatPage（AuthGuard 保护）
├── /:sessionId     → ChatPage（AuthGuard 保护）
└── *               → Navigate to="/"
```

启动时调 `authStore.init()`（从 localStorage 恢复 token），监听 `auth:unauthorized` 事件 → `logout()`。

### 4. API 层

#### `src/api/client.ts`
- fetch 封装，支持 GET / POST / PATCH / DELETE
- 自动注入 `Authorization: Bearer {token}`
- 401 时广播 `auth:unauthorized` 自定义 DOM 事件
- 支持 204 No Content（返回 `undefined`）
- 错误时抛出 `ApiError(status, message)`

#### `src/api/auth.ts`
- `loginAPI(password)` → POST `/auth/login` → `{ token, expires_at }`

#### `src/api/sessions.ts`
- `fetchSessionsAPI()` → GET `/sessions`，**解包 `{ sessions: [...] }` 包裹对象**（曾是重要 Bug 修复点）
- `createSessionAPI(scene_type, model)` → POST `/sessions`
- `deleteSessionAPI(id)` → DELETE `/sessions/{id}`
- `updateSessionAPI(id, data)` → PATCH `/sessions/{id}`，`data` 类型为 `Partial<Pick<Session, 'title' | 'model' | 'scene_type'>>`

#### `src/api/chat.ts`
- `fetchMessagesAPI(sessionId)` → GET `/sessions/{id}/messages`
- `streamChat(sessionId, model, content, token)` → POST `/v1/chat/completions`，返回原始 `Response`

### 5. 状态管理

#### `authStore.ts`
- `token / isLoggedIn`
- `init()`：从 localStorage 恢复
- `login(password)`：调 API，存 token
- `logout()`：清 token，isLoggedIn = false → AuthGuard 触发跳转

#### `sessionStore.ts`
- `sessions[] / currentSession / loading`
- `fetchSessions()`：拉列表，按 `updated_at` 倒序排列
- `createSession(scene_type, model)`：创建并自动选中
- `selectSession(id)`：切换当前会话
- `deleteSession(id)`：删除，自动选中第一条
- `updateSessionModel(model)`：**先 PATCH 后端 → 再 fetchSessions → 再定位 fresh session** 赋给 `currentSession`（避免本地展开覆盖其他字段如 title）
- `getGroup(dateStr)`：日期 → today / yesterday / previous
- `formatSessionTime(dateStr)`：格式化显示时间

#### `chatStore.ts`
- `messages[] / isStreaming / currentThinking / currentText`
- `loadMessages(sessionId)`：加载历史，兼容 `{ messages: [] }` 包裹格式，records 从后端倒序到正序 reverse，每条 record 展开为 user + assistant 两条 ChatMessage
- `sendMessage(sessionId, model, content)`：
  1. 本地立即添加用户消息
  2. **自动命名**：title 为空或 `'新对话'` 时，取内容前 20 字（超出加 `…`）静默异步更新标题
  3. 调 `streamChat` 发起流式请求
  4. 逐行解析 SSE（`data: {...}`）
  5. `thinking_delta` → 追加 currentThinking
  6. `text_delta` → 追加 currentText
  7. `done` → 组装 assistantMsg，清空流式状态
  8. finally 安全兜底：强制 `isStreaming = false`
- `clearMessages()`：切换 session 时清空

## 四、易错点 / 开发陷阱备忘

### API 数据格式
- **GET /sessions** 返回 `{ sessions: [...] }` 包裹对象，不是直接数组。`fetchSessionsAPI` 需要 `res.sessions`。
- **GET /sessions/:id/messages** 返回 `{ messages: [...], page, page_size }`，每条 record 含 `user_msg` + `assistant_msg` 两个字段，需要展开成两条 ChatMessage，且后端是最新在前（需 reverse）。

### 状态更新顺序陷阱
- `updateSessionModel` 不能用本地展开 `{ ...session, model }` 赋值，会把 title 等字段恢复成旧值。必须先 PATCH 后端，再 fetchSessions，再用后端返回的数据更新 currentSession。
- 自动命名（chatStore）与模型切换（sessionStore）都会修改 session，需要保证都以后端为准，不在前端本地合并。

### 闪烁问题模式
- 任何"先清空再异步加载"的操作（clearMessages + loadMessages）都会导致 UI 在加载期间短暂显示空态（欢迎页）。解决方案：引入 loading 标志位，在 loading 期间保持之前的 UI 状态。

### useEffect 依赖
- `fetchSessions` 应依赖 `token` 变化（`[token, fetchSessions]`），避免 token 异步恢复后会话列表不刷新。
- 点击外部关闭面板的 mousedown 监听：只在面板打开时注册（`if (!showSceneSelect) return`），关闭时 cleanup 移除，避免内存泄漏和重复注册。

### zustand 跨 store 调用
- 在 `chatStore` 的 `sendMessage` 中需要访问 `sessionStore` 的数据，使用 `useSessionStore.getState()` 获取实例（不能在 store 内部 hook 方式调用），然后调用其方法。

### 重命名 input 的 onBlur 陷阱
- 重命名 input 的 `onBlur` 触发 `handleRenameConfirm`，`handleRenameConfirm` 内部有空标题检查（空时不保存，直接清空编辑态）。注意 Escape 应该先 `setEditingId(null)` 再让 blur 自然触发，但 Escape 的 keyDown 里要先 `setEditingId(null)` 避免 blur 再次调用 confirm 产生不必要的 API 请求。

### 场景面板点击外部关闭
- `sceneRef` 挂在场景网格 `<div>` 上，但"+" 按钮本身不在 ref 内。所以点 "+" 按钮切换时，mousedown 事件先触发（关闭面板），再触发 click（重新打开）。这是正确行为，结果是点 "+" 切换开关。如果将来要把 "+" 按钮也纳入 ref 范围，需要把整个顶部 div 都包进去。
