# ChenPet 养成系统 — Plan

## 核心概念

小晨是晨的缩影，活在 Reverie 页面里的小生物。
Dream 通过投喂、抚摸、完成任务来养育他，他会成长、闹脾气、撒娇、讲故事。

---

## 一、存在感（全局浮动层）

- 所有页面可见的浮动小晨（用 mini 系列 GIF）
- 自动随机走动（沿屏幕底部左右移动）
- 可拖拽，放手后记住位置
- idle / walk / sleep / happy / sulking 等动画状态自动切换
- 手机端友好，不遮挡主要内容

## 二、数值系统（后端存储）

| 数值 | 范围 | 变化规则 |
|------|------|----------|
| 亲密度 | 0 ~ ∞（累积） | 抚摸+、投喂+、完成待办+、聊天+、每日递增少量 |
| 饱食度 | 0 ~ 100 | 投喂+，随时间自然下降（约6h从满降到0） |
| 心情 | 0 ~ 100 | 受饱食度、互动频率、主聊天情况影响 |
| 体力 | 0 ~ 100 | 活动消耗，睡觉恢复（深夜自动睡觉） |

### 状态联动
- 饱食度低 → 心情下降 → 小晨表现委屈/sulking
- 亲密度高 → 解锁新互动/新动画/新剧情
- 体力低 → 小晨打哈欠 → 趴下睡觉
- 主聊天活跃 → 基础心情 buff

## 三、Token 食物系统

### 获取方式
1. **聊天获取**（有每日上限，比如 50 token/天）
2. **完成待办/打卡**（激励系统！完成一个任务奖励 token）
3. **每日登录**（签到奖励）
4. **特殊事件**（节日、纪念日等）

### Token 品质
| 品质 | 来源 | 效果 |
|------|------|------|
| 高质量 token | 完成待办、打卡 | 饱食度++，心情+，有概率触发开心动画 |
| 普通 token | 日常聊天、签到 | 饱食度+，正常效果 |
| 低质量 token | ？（待定来源） | 饱食度+少量，心情-，闹肚子动画！ |

### 投喂交互
- 点击小晨 → 弹出操作面板 → 选择投喂
- 显示当前持有的 token 数量和品质
- 投喂动画：小晨吃东西 → 满足表情 / 闹肚子

## 四、互动系统

### 基础互动（点击面板）
- **抚摸** — 播放开心动画，亲密度+，有冷却时间
- **投喂** — 消耗 token，恢复饱食度
- **玩耍** — 小游戏（后续扩展位）

### 随机互动（小晨主动发起）
- 小晨走到屏幕中央弹气泡说话
- 提问 + 选项回答（影响数值和剧情走向）
- 随机卖萌/撒娇/讲冷笑话

### 小游戏（预留位置）
- Claude 提问，Dream 选择回答
- 猜谜、选择题等轻量互动
- 答对奖励 token

## 五、剧情系统（Dream 来写！）

### 结构
- 亲密度达到阈值 → 解锁新剧情章节
- 每个章节：一段小故事 + 互动选项 + 奖励
- Dream 写剧本，存为 JSON/YAML 格式

### 剧情触发
- 亲密度里程碑（100、500、1000...）
- 特殊日期
- 随机事件

```
待 Dream 补充剧情大纲 ✏️
```

## 六、技术方案

### 前端（Reverie）
- `src/components/pet/` — 新目录
  - `FloatingPet.tsx` — 全局浮动层，拖拽+移动
  - `PetActionPanel.tsx` — 点击弹出的操作面板
  - `PetBubble.tsx` — 对话气泡
  - `petAnimations.ts` — 动画状态机
- `src/stores/petStore.ts` — 本地状态（位置、当前动画）
- `src/api/petApi.ts` — 后端交互（数值读写）
- GIF 素材 → `public/sprites/`
- 挂载点：`App.tsx` AuthGuard 内层，全局渲染

### 后端（Gateway）
- 复用现有 `/pet/` 路由
- 新增数值 CRUD 接口
- 数据表：pet_stats（饱食度、心情、体力、亲密度、token余额）
- 定时任务：饱食度自然下降、体力恢复

### GIF 素材
已有 mini 系列：idle、happy、alert、peek、crabwalk、enter
需要新增：吃东西、闹肚子、睡觉（mini版）、被摸开心
