## 一、骰子 + 检定系统

### 核心机制

**骰子自适应缩放**：不固定面数，随剧情难度动态升级。

**项目级配置（存在 character_state 里）：**
```json
{
  "dice_config": {
    "base_die": 10,        // 初始骰子面数（d10）
    "current_die": 10,     // 当前骰子面数（GM 可升级）
    "crit_success": "max", // 投出最大值 = 大成功
    "crit_fail": 1         // 投出 1 = 大失败
  }
}
```

GM（晨）可以在剧情推进中升级骰子：`[骰子：d20]`——前端解析后更新配置，从此检定用 d20。这样随着故事难度升级，骰子跟着升，目标数也可以更高（比如目标 15、20），但玩家的属性和装备也在成长。

**大成功 / 大失败：**
- **大成功**：投出骰子最大值（d10 投 10，d20 投 20，d24 投 24）——无条件成功，效果超乎想象
- **大失败**：投出 1——无条件失败，灾难性后果
- 这两个结果无视属性加成，纯看运气

**检定流程：**
1. 晨（GM）叙事 → 发起检定：`[检定：属性名·目标数]`
   - 例：`[检定：剑术·7]`、`[检定：社交·12]`
   - 目标数由 GM 根据剧情难度自由决定（不再用"简单/困难"等级，更灵活）
2. 前端显示检定气泡：属性名、当前属性值、装备加成、目标数、投骰按钮
3. Dream 点击投骰
4. 计算：**d(当前骰) + 属性值 + 装备加成 vs 目标数**
5. 结果气泡：成功 / 失败 / 大成功 / 大失败
6. 结果注入晨的上下文，晨继续叙事

### 属性点分配系统

创建角色时：
- 系统给一个**总属性点池**（比如 20 点）
- 玩家自由分配到各属性上
- 属性列表由项目自定义（不同故事不同属性名）
- 分配完后可以随时在角色卡里调整（前期允许重分配）

### 自由投骰

输入框旁边的骰子按钮，Dream 随时可以投一个当前面数的骰子，用于叙事分支决策。

### 前端气泡设计

**检定待投气泡**（半透明暗色底，和 RP 背景融合）：
```
┌───────────────────────────────────┐
│  ⚔ 剑术检定                       │
│                                    │
│  属性 4 ＋ 长剑 +1 ＝ 基础 5       │
│  目标 ≥ 7    需投 ≥ 2              │
│                                    │
│          [ 🎲 投骰 ]               │
└───────────────────────────────────┘
```

**结果气泡**：
```
┌───────────────────────────────────┐
│  🎲 d10 → 8                       │
│  8 + 剑术(4) + 长剑(+1) = 13      │
│  目标 7 ─── ✦ 成功                 │
└───────────────────────────────────┘
```

**大成功**（特殊金色/光效动画）：
```
┌───────────────────────────────────┐
│  🎲 d10 → 10 ！                   │
│  ✦✦ 大成功 ✦✦                     │
└───────────────────────────────────┘
```

**大失败**（暗红/碎裂效果）：
```
┌───────────────────────────────────┐
│  🎲 d10 → 1                       │
│  ✘ 大失败                          │
└───────────────────────────────────┘
```

## 二、角色状态面板（Character Sheet）

### 数据结构

```json
{
  "name": "伊西多（棘刺）",
  "hp": { "current": 10, "max": 10 },
  "currency": { "name": "金币", "amount": 6 },
  "total_points": 20,
  "attributes": {
    "剑术": 4,
    "炼金": 5,
    "社交": 2,
    "感知": 3,
    "体魄": 4,
    "意志": 3
  },
  "inventory": [
    { "name": "花纹匕首", "description": "尾部有独特花纹图案", "stat_bonus": null },
    { "name": "长剑", "description": "标准伊比利亚骑士剑", "stat_bonus": { "剑术": 1 } },
    { "name": "铜奢靡卡", "description": "苏丹的游戏·七天内折断", "stat_bonus": null }
  ],
  "status_effects": [],
  "dice_config": {
    "base_die": 10,
    "current_die": 10,
    "crit_success": "max",
    "crit_fail": 1
  }
}
```

### 前端 UI

**项目详情页 → 角色卡标签**：
- 角色名 + 可选头像
- HP 条（视觉化，暗红色调）
- 金币/货币显示
- 属性列表（可编辑数值，总点数校验）
- 物品栏（增删改，可设装备加成属性）
- 当前骰子：d10/d20 等

**聊天页面 → 顶部迷你面板（可折叠）**：
- 一行显示：HP ❤ 10/10 · 💰 6 · 🎲 d10
- 点击展开详细属性

### 状态变更语法

晨在 RP 叙事中触发（前端解析，不显示原文，显示为变更气泡）：
```
[状态：金币 -3]
[状态：HP -2]
[状态：物品 +花纹匕首（尾部有独特花纹图案）]
[状态：物品 -长剑]
[状态：属性 社交+1]
[骰子：d20]
```

变更气泡样式（小型，嵌在对话流中）：
```
  ┌─────────────────┐
  │ 💰 6 → 3        │
  └─────────────────┘
```

## 三、晨的回复分层渲染

剧本模式下，晨的一段回复会被解析成多个视觉块：

| 内容类型 | 识别方式 | 渲染样式 |
|---------|---------|---------|
| GM 旁白/场景 | *斜体文本*（markdown italic） | 全宽文字块，衬线字体，较大行距，略透明底色，左侧细竖线 |
| NPC 对话 | 【角色名】"对话内容" | 角色名标签（小圆角 badge）+ 对话气泡，不同 NPC 可能不同重点色 |
| 角色动作 | **加粗** 或普通描述 | 跟 GM 旁白类似但无竖线 |
| 检定标记 | `[检定：属性·目标]` | 检定交互气泡（见上） |
| 状态变更 | `[状态：...]` | 变更小气泡 |
| 骰子升级 | `[骰子：dN]` | 系统提示气泡："骰子升级为 dN" |
| 笔记标记 | `[笔记：类型｜内容]` | 不在对话流显示，静默写入故事笔记 |

Dream 的消息保持原样（作为角色行动/对话的输入）。

## 四、剧情笔记 / 故事时间线

### 数据

新表 `project_notes`：
- id, project_id, chapter (int), content, note_type (event/foreshadow/npc/loot), auto (bool), created_at

### 前端

项目详情页 → "笔记"标签：
- 按章节分组的时间线

- 可手动增删改
- 晨自动记录的标 "auto" 徽章

## 五、RP 模式整体前端改造

### 进入剧本模式时的 UI 变化

当 session.scene_type === "roleplay"：

1. **背景切换**：从 Reverie 日常暖色调切换到暗色沉浸式背景
   - 项目可配置背景主题（海洋/沙漠/城堡/森林/自定义图片）
   - 半透明雾气/粒子效果（可选）

2. **消息区域**：
   - 气泡半透明，融入暗色背景
   - 晨的消息分段渲染（GM/NPC/系统）
   - 检定/状态变更气泡穿插在叙事中

3. **顶部**：
   - 项目名 + 章节标题
   - 迷你角色状态条

4. **输入区域**：
   - 增加骰子按钮
   - 其他保持不变

5. **侧边**：
   - 角色卡快捷入口
   - 故事笔记快捷入口

## 七、已确认的设计决策（Dream 2026-05-24）

1. ✅ 骰子面数不由玩家选择，GM 决定何时升级（d10 → d20 → d24 等）
2. ✅ 大成功 = 投出最大值，大失败 = 投出 1，无视属性加成
3. ✅ 属性点给总数，玩家自由分配
4. ✅ 视觉风格：暗色沉浸式（参考「献给海的狂言」+ 泊屿聊天界面）
5. ✅ 骰子/检定/状态变更都用特殊气泡，晨只负责叙述
6. ✅ GM 旁白 vs NPC 对话 vs 系统消息 视觉分层
7. ✅ 检定结果气泡显示数据，晨的后续叙事描述故事表现
